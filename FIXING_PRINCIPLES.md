# 代码修复原则检查清单

## 核心原则

在修复任何代码问题时，必须严格遵守以下原则：

### ✅ 原则 1: 不改变模型架构
- [ ] 不修改任何模型类定义（如 `nn.Module` 子类）
- [ ] 不修改网络层结构、通道数、卷积核大小等
- [ ] 不修改注意力机制、残差连接等架构组件
- [ ] 不修改模型初始化参数

**允许的修改**：
- 添加异常处理包装现有模型调用
- 添加设备转换（CPU/GPU）但不改变计算逻辑
- 添加可选依赖的导入处理

### ✅ 原则 2: 不改变计算逻辑
- [ ] 不修改前向传播（forward）函数的核心计算
- [ ] 不修改损失函数计算
- [ ] 不修改优化器更新逻辑
- [ ] 不修改梯度计算
- 不修改任何数学运算（矩阵乘法、卷积、归一化等）

**允许的修改**：
- 添加数值稳定性检查（如 NaN/Inf 检查）
- 添加设备兼容性处理（CPU/GPU 转换）
- 添加数据类型转换（但结果必须相同）

### ✅ 原则 3: 不改变训练/推理流程
- [ ] 不修改训练循环结构
- [ ] 不修改推理采样流程
- [ ] 不修改数据加载顺序
- [ ] 不修改 checkpoint 保存/加载逻辑（除非是修复加载错误）

**允许的修改**：
- 添加文件存在性检查
- 添加异常处理，避免流程中断
- 添加日志输出（不影响流程）

### ✅ 原则 4: 只增加错误处理和兼容性
- [ ] 所有修改都是添加性的（添加 try-except、检查等）
- [ ] 不删除或修改原有功能代码
- [ ] 只添加兼容性处理（可选导入、回退机制等）

**允许的修改类型**：
- 环境兼容性修复（网络解析、文件系统等）
- 可选依赖处理（缺失模块的可选导入）
- 文件/模型加载健壮性（文件存在性检查、回退路径）
- 异常处理增强（try-except 包装）
- 导入路径修正（但功能不变）

### ✅ 原则 5: 所有修改都是可逆的
- [ ] 修改可以通过注释/取消注释轻松恢复
- [ ] 修改不破坏原有代码结构
- [ ] 如果原始环境完整，修改可以移除而不影响功能

## 修复前检查清单

在进行任何修复前，请确认：

1. **问题识别**
   - [ ] 问题是否属于环境兼容性问题？
   - [ ] 问题是否属于缺失依赖问题？
   - [ ] 问题是否属于运行时错误（而非逻辑错误）？

2. **修改方案评估**
   - [ ] 修改是否只添加错误处理/兼容性代码？
   - [ ] 修改是否不触及模型定义？
   - [ ] 修改是否不改变计算逻辑？
   - [ ] 修改是否可逆？

3. **影响范围评估**
   - [ ] 修改是否只影响错误处理路径？
   - [ ] 正常执行路径是否完全不变？
   - [ ] 模型输出是否保持一致？

## 修复后验证

修复完成后，请确认：

1. **功能验证**
   - [ ] 原有功能仍然正常工作
   - [ ] 模型输出结果与修复前一致（如果修复前能运行）
   - [ ] 训练/推理流程完整执行

2. **代码质量**
   - [ ] 添加了适当的注释说明修改原因
   - [ ] 异常处理有清晰的错误信息
   - [ ] 代码符合项目风格

3. **文档更新**
   - [ ] 在 `MODIFICATION_LOG.md` 中记录修改
   - [ ] 说明修改原因、位置和影响

## 禁止的修改类型

❌ **绝对禁止**：
- 修改模型架构（层数、通道数、结构）
- 修改损失函数公式
- 修改优化器配置
- 修改训练/推理的核心算法
- 删除或重写原有功能代码
- 改变模型权重加载方式（除非修复加载错误）

## 允许的修改类型

✅ **允许**：
- 添加 try-except 异常处理
- 添加文件/路径存在性检查
- 添加可选依赖的导入处理
- 添加设备转换（CPU ↔ GPU）
- 添加回退机制（如文件不存在时的替代路径）
- 修正导入路径错误
- 添加缺失的工具函数（不影响模型逻辑）
- 添加日志和调试信息

## 示例：正确的修复方式

### ✅ 正确示例 1: 添加异常处理
```python
# 修改前
hostname = socket.gethostbyname(socket.getfqdn())

# 修改后
try:
    hostname = socket.gethostbyname(socket.getfqdn())
except socket.gaierror:
    hostname = "localhost"  # 回退方案
```

### ✅ 正确示例 2: 可选依赖导入
```python
# 修改前
from tacm.modules import LPIPS

# 修改后
try:
    from tacm.modules import LPIPS
except ImportError:
    LPIPS = None  # 可选依赖
```

### ✅ 正确示例 3: 设备转换（不改变计算）
```python
# 修改前
c_temp = BEATs_model.extract_features(audio, padding_mask=None)[0]

# 修改后
BEATs_model_cpu = BEATs_model.cpu()
c_temp = BEATs_model_cpu.extract_features(audio, padding_mask=None)[0]
BEATs_model.to(dist_util.dev())
c_temp = c_temp.to(dist_util.dev())
# 注意：计算结果完全相同，只是改变了执行设备
```

## 项目完整性保证

**本项目仍然属于原项目模型**。所有修改都是技术性的兼容性修复，不涉及模型架构、训练逻辑或推理计算的改变。这些修改使得项目能够在当前环境中正常运行，同时保持模型行为的完全一致性。
